How to use your icons

Websites & PWAs

Drop the PNGs in your site’s /public (or root) folder and add these to your <head>: 

<!-- Favicon basics -->
<link rel="icon" href="/favicon-32.png" sizes="32x32" type="image/png">
<link rel="icon" href="/favicon-16.png" sizes="16x16" type="image/png">

<!-- Apple touch icon (iOS home screen) -->
<link rel="apple-touch-icon" href="/apple-touch-180.png" sizes="180x180">

<!-- Android/Chrome PWA -->
<link rel="manifest" href="/manifest.json">
<meta name="theme-color" content="#111827">

Suggested sizes: 16, 32, 48, 64, 128, 180, 192, 512.

Sample manifest.json

Save as /manifest.json and tweak names/paths/colors: 

{
  "name": "Your App Name",
  "short_name": "YourApp",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#111827",
  "theme_color": "#111827",
  "icons": [
    { "src": "/android-chrome-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/pwa-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any maskable" }
  ]
}

If you include a maskable 512×512, Android crops nicely for different shapes.

iOS / iPadOS (Xcode)

  1. In Xcode, open Assets.xcassets → AppIcon.
  2. Drag the generated PNGs (from the iOS pack) into the matching slots.
  3. Set App Icons Source in your target → General to your AppIcon set.
Apple HIG: App Icons

Android (Adaptive + Launcher)

  1. Place PNGs into the corresponding mipmap-* folders under app/src/main/res/.
  2. For Adaptive Icons, provide foreground/background layers (or color).
  3. Set android:icon and android:roundIcon in AndroidManifest.xml.
